# AccessFox Demo Recording Guide

## Setup (2 minutes)

1. **Start the app:**
   ```bash
   cd /Users/dbaikov/projects/hackathons/bright-data-hack/accessfox
   npm run dev
   ```
2. **Open browser** → http://localhost:3000
3. **Set browser window** to 1280×800 or full screen. Hide bookmarks bar.
4. **Open narration audio** in QuickTime Player: `submission/narration.aiff` — don't play yet.
5. **Open macOS Screen Recording:**
   - Press `Cmd + Shift + 5`
   - Select "Record Entire Screen" or a window
   - Enable microphone if recording your own voice, or skip if using the audio file separately

---

## Recording Steps

### Step 1 — Show the homepage (0:00–0:07)
- App is open at http://localhost:3000
- Visible: **AccessFox** logo, **Healthcare account-readiness intelligence**, search bar
- Start screen recording + start narration audio simultaneously

### Step 2 — Show search examples (0:07–0:20)
- Slowly move cursor over the example terms below the search bar:
  `theranostics · cell therapy · proton therapy · robotic surgery · hospital-at-home`
- Clear the existing query text

### Step 3 — Enter search term and click (0:20–0:28)
- Type: `radiopharmaceutical therapy / theranostics adoption readiness`
- Pause 1 second
- Click **Find Expansion Opportunities**

### Step 4 — Show loading → results + badge (0:28–0:38)
- Let loading skeletons show for a moment
- When results appear, move cursor to the **"Live Web Signals via Bright Data"** green badge
- Pause on it for 2 seconds

### Step 5 — Scroll through ranked cards (0:38–0:55)
- Slowly scroll down through the cards
- Pause on card #1 — move cursor to the **Account Readiness Score** ring

### Step 6 — Highlight Evidence Ledger (0:55–1:05)
- Scroll to show the **Evidence Ledger** section of the top card
- Move cursor slowly over 2–3 evidence items with source links

### Step 7 — Show Why Now + Recommended Action (1:05–1:18)
- Scroll slightly to show **Why Now**, **Recommended Action**, **Outreach Angle** fields
- Move cursor over each label

### Step 8 — Scroll back to search bar (1:18–1:28)
- Scroll back to top
- Hover over example terms: `cell therapy`, `proton therapy`, `robotic surgery`

### Step 9 — End frame (1:28–1:35)
- Leave cursor on the **AccessFox** header
- Stop recording

## Export

1. Stop screen recording → save as MP4
2. If you recorded voice separately, combine in iMovie or QuickTime:
   - File → New Movie
   - Drag screen recording + narration.aiff
   - Align to start
   - Export → File → 1080p MP4
3. Save final file as: `submission/accessfox-demo.mp4`

---

## Tips

- Record at least twice — first take is usually too fast
- Speak at ~160 words/minute — slower than you think
- The narration audio file at `submission/narration.aiff` is ~85 seconds — match your pacing to it
- If using live mode (Bright Data key active), results take ~15s — budget for that in timing
- If using demo mode (no keys), results are instant — you can pad with slower scrolling
